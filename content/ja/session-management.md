# 11. セッション＆コンテキスト管理

> Claude Codeを長く、効率的に使う方法。長くなったセッションの圧縮、前回の作業の再開、複雑な作業前の安全策まで。

---

## コンテキストウィンドウとは？

**背景知識のない方へ**：

Claudeは会話するとき、これまでのやり取りの内容をすべて覚えながら作業しています。しかし、この記憶には限界があるため、会話が積み重なっていくと、古い内容から押し出されていきます。

「さっき確かに言ったのに、なぜ分からないの？」という状況がまさにこれです。Claudeがわざと忘れたのではなく、メモ帳がいっぱいになって古い内容が切り落とされたのです。

---

**背景知識のある方へ**：コンテキストウィンドウは、AIモデルが一度に処理できるトークンの最大数です。Claude Codeはスライディングウィンドウ方式で動作し、上限を超えると古いトークンから切り落とされます。

---

**共通して起こること**：

```
会話が長くなるにつれて：
  序盤の内容が押し出され始める
  → Claudeが前に行った作業を覚えていないことがある
  → 回答の品質が低下し始める
  → 同じミスを繰り返し始める
```

そのまま放置すると、会話がどんどん混乱していきます。たまに整理してあげる必要があります。

---

## /compact — 会話の圧縮

```
/compact
```

これまでの会話を要約してコンテキストを削減します。

- 重要な内容は要約に含まれます
- セッションを切らずにそのまま続けられます
- **使うタイミング**：会話が長くなったと感じたとき、回答がおかしくなり始めたとき

圧縮の方向性を直接指定することもできます：

```
/compact リサーチ結果と主な発見事項を中心に要約してください
/compact 作成中のレポートの構成と進捗状況を中心に
/compact 結論/根拠/未解決の質問だけ残してください
```

---

## セッション再開の方法

### --continue — 直前のセッションを引き継ぐ

```bash
claude --continue
```

最後に終了したセッションをそのまま引き継ぎます。

- ターミナルを閉じて再度開いたとき
- 途中の作業を翌日続けるとき

### --resume — セッションを選択して再開

```bash
claude --resume
```

以前のセッション一覧を表示し、選択して引き継ぎます。

- 複数のプロジェクトを交互に作業するとき
- 数日前の特定のセッションに戻りたいとき

---

## いつ新しいセッションを始めるべきか

| 状況 | 推奨 |
|------|------|
| まったく別のテーマ/プロジェクト | 新しいセッション |
| Claudeが前の内容を忘れ始めた | 新しいセッション + 重要なコンテキストを再提供 |
| 同じエラーが繰り返される | 新しいセッション（コンテキストが汚染されている可能性） |
| 会話が長くなったが同じ作業 | /compact後に続行 |
| 途中の作業を翌日続ける | --continue |

## 複雑な作業の前にはPlan Mode

複数のファイルを同時に修正したり、大きな変更が予想されるとき、実行前にまず計画を確認してもらうことができます。

```
Shift + Tab  →  Plan Modeに切り替え
```

Plan ModeでのClaudeは：
1. 実行計画のみを表示します（ファイルは変更しません）
2. 「このように進めようと思いますが、よろしいですか？」と先に確認を取ります
3. 承認を得てから実行します

**使うタイミング**：複数ファイルの同時修正、フォルダ構造の再編、大量の内容置換、元に戻しにくい作業の前

---

## 並列セッション

ターミナルを複数開いて、同時に別々の作業を任せることができます。

```
ターミナル1: claude（競合A社のリサーチ）
ターミナル2: claude（レポート草案の作成）
ターミナル3: claude（プレゼン資料の整理）
```

各セッションは独立して動作します。

> **注意**：同じファイルを2つのセッションで同時に修正すると競合が発生します。担当ファイル/役割を分けて使いましょう。

---

## よくある状況別の対処法

### Claudeの回答が急におかしくなったとき

1. `/compact` で圧縮を試みる
2. それでもおかしければ → 新しいセッションを開始
3. 新しいセッションで重要なコンテキストだけ再提供：`「現在[プロジェクト名]の作業中で、ここまで[要約]を行った。次に[作業]を行う必要がある。」`

### 長い作業を途中で止めなければならないとき

```
作業中 → Ctrl+Cで中断
→ ターミナルを閉じる
→ 次回: claude --continue
```

Claudeは中断された地点からの状態を覚えています。

---

> **コストにも関係します**：コンテキストが長いほど多くのトークンを消費します。/compactを定期的に使えば、速度も上がりコストも削減されます。→ 詳しくはセクション18のコスト管理をご覧ください。
