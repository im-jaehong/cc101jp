# 12. GitHub入門

> 🖥️ このセクションは主に**コードを書く開発者**向けの内容です。コーディング作業がない場合は、下記の**非開発者向けファイルバージョン管理**のヒントだけ読んで次のセクションに進んでも構いません。コマンドを知らなくても大丈夫です — 自然言語から始められます。

---

## 非開発者の方へ：ファイルバージョン管理の3つの方法

コーディングをしなくても、ファイルが誤って変更されたり消えてしまう状況は誰にでも起こります。簡単な代替手段をご紹介します：

| 状況 | 方法 |
|------|------|
| **ドキュメントのバージョン記録** | Google Docs・Notionの「バージョン履歴」機能 — いつでも以前のバージョンに復元可能 |
| **作業前のスナップショット** | `「reports/フォルダをreports_backup_今日の日付でコピーして」`（Claude Codeに言えばOK） |
| **直前の作業を取り消し** | `Esc Esc` — Claude Codeセッション中、直前の作業を即座に元に戻します |

> **Gitに興味が出てきましたか？** Gitはコード以外にも、あらゆるファイルのバージョン管理が可能です。リサーチファイル、レポート、企画書もGitで安全に管理できます。興味があれば、このセクションを読み進めてください。

---

## なぜGit & GitHubが必要なのか？

Claude Codeでコードを修正していると、ファイルが次々と変わっていきます。Gitがないと：

- 昨日うまく動いていたコードが今日なぜ動かないのか分からない
- Claudeが誤って重要なコードを削除しても元に戻せない
- チームメンバーと同じファイルを修正して、お互いの作業を上書きしてしまう

**Git**：すべての変更履歴を保存し、いつでも以前の状態に戻せるツール
**GitHub**：その履歴をクラウドにバックアップし、チームと共有するプラットフォーム

Claude Codeユーザーにとって、Git + GitHubは必須です。

---

## Google DriveとGitのたった一つの違い

Gitを初めて使う方が最も混乱するポイントです。

| | Google Drive | Git |
|--|-------------|-----|
| ファイルを修正すると | 自動的にクラウドにアップされる | **自分で保存してアップする必要がある** |

**この違いだけ覚えておけば大丈夫です。**

Gitには常に2つのステップがあります：

1. 保存（Commit）— 変更内容を自分のパソコンに記録
2. アップロード（Push）— 記録した内容をGitHubにアップロード

> **なぜ2ステップなのか？** インターネットがなくてもローカルで保存でき、後から準備ができたものだけ選んでアップロードできます。

## 最初の一歩：git-teacherプラグインで5分セットアップ

Gitコマンドを覚える必要はありません。**git-teacher**プラグインが自然言語ですべて処理してくれます。

gptaku_pluginsをセクション04でインストール済みであれば、すでに準備完了です。まだの場合は：

```shell
/plugin marketplace add https://github.com/fivetaku/gptaku_plugins.git
/plugin install git-teacher
```

インストール後、Claude Codeを完全に終了してから再起動してください。

### 初回のみ：セットアップ

```
「Gitを始めて」
```

自動的に処理されること：

1. Gitのインストール確認（未インストールの場合はインストール案内）
2. GitHub CLI(gh)のインストール確認
3. GitHubアカウントへのログイン（ブラウザが自動で開きます）
4. プロジェクトフォルダの設定
   - **新規プロジェクト** → 新しいフォルダ + GitHubリポジトリを作成
   - **既存プロジェクト** → GitHubからクローン
   - **現在のフォルダ** → 現在の場所をリポジトリとして設定

すでに完了しているステップは自動的にスキップされます。

## 毎日行う3つのこと

### 1. 保存する（Commit）

```
「保存して」
```

変更されたファイルの一覧を表示し、説明（コミットメッセージ）の入力を求めます。

```
例：「ログインページのデザイン修正」
    「バグ修正：パスワード検証エラー」
```

保存完了後、以下の案内が表示されます：

```
✅ 保存完了！
まだ自分のパソコンにしかありません。
GitHubにアップするには「アップして」と言ってください。
```

### 2. GitHubにアップロードする（Push）

```
「アップして」
```

保存した内容をGitHubにアップロードします。完了するとリポジトリのリンクが表示されます。

自動エラー処理：
- チームメンバーが先にアップした内容と重複する場合、自動的にマージしてアップ（`--rebase`）
- SSH認証エラーの場合はHTTPSに自動切り替え

### 3. レビューを依頼する（Pull Request）

チームメンバーに自分の変更をレビューしてもらいたいとき：

```
「レビュー依頼して」
```

自動的に処理：Branch作成 → 保存＆アップロード → PR作成 → リンク提供 → 元の状態に復帰

```
やりたいこと                      こう言ってください
─────────────────────────────────────────────────
初期設定                         「Gitを始めて」
現在の変更ファイル確認              「今どんな状態？」
                                「何が変わった？」
保存する（Commit）                「保存して」
                                「セーブして」

                                「アップロードして」
レビューを依頼する（PR）           「レビュー依頼して」
                                「チームに見せたい」
```

| 「アップして」 | `/git-teacher アップロード` |
| 「レビュー依頼して」 | `/git-teacher レビュー` |

## Git用語クイックガイド

初めて見る単語が出てきたときの参考にしてください。

| Git用語 | 分かりやすい言葉 | たとえ |
|----------|--------|------|
| **Repository** | プロジェクトの保管庫 | GitHubの共有フォルダ |
| **Commit** | 保存する | ファイルを梱包してラベルを貼る |
| **Push** | アップロードする | 梱包したファイルをGitHubに送る |
| **Pull** | 取得する | GitHubから最新の内容を受け取る |
| **Branch** | 作業スペース | 原本に触れない個人の作業室 |
| **Pull Request** | レビュー依頼 | Google Docsの「編集の提案」モード |
| **Merge** | 統合する | 編集の提案を原本に反映 |
| **Clone** | コピーして取得する | 共有フォルダを自分のパソコンにダウンロード |

分からない用語があればいつでも聞いてください：

```
「pushとcommitの違いは何？」
```

## Claude Codeと一緒に過ごす一日のルーティン

```
作業開始：
  claude起動 → 「Gitを始めて」（初日のみ）

作業中：
  コード修正 → 「保存して」 → 「アップして」
  （大きな作業の前に：「今どんな状態？」で確認）

作業終了：
  「保存して」 → 「アップして」（必ず両方のステップ）

チーム協業：
  機能完成 → 「レビュー依頼して」 → PRリンクを共有
```

このルーティンを守れば、Claude Codeで作業した内容が常に安全に保管されます。誤ってコードが壊れても、いつでも以前の状態に戻ることができます。

## .gitignore — APIキーがGitHubにアップされないように

Gitを初めて使うときに最もよくあるミスが、`.env` ファイルに含まれるAPIキーをGitHubにアップロードしてしまうことです。

git-teacherプラグインは「保存して」の際にプロジェクトタイプを自動検出して `.gitignore` を生成します。すでに存在する場合はスキップします。

自分で作成したい場合は：

```
# .gitignore の例
.env
.env.local
node_modules/
.DS_Store
```

---

> **Gitコマンドを直接使いたい場合**：Claude Codeにそのまま言えばOKです — `git commit`、`git push` を直接実行してくれます。git-teacherプラグインは、非開発者向けのガイダンス、自動エラー処理、日本語でのステータス説明が追加されたレイヤーです。Gitに慣れていない方はgit-teacherから始めてみてください。
