# 18. 自分だけのAIワークスペースを設計する

> CLAUDE.md、Skills、Hooksを学んだら、次は組み合わせる番です。単なるAIツールではなく、自分のための自動化オフィスを作る方法。

---

## ワークスペースとは？

**一般的なAIの使い方**：「質問 → 回答」（毎回ゼロから）

**ワークスペース**：役割が決められたAIが、自分だけのルールで、一貫した方式で動作するシステム

構成要素：

```
CLAUDE.md     → AIの役割、ルール、知識（長期記憶）
フォルダ構造   → 成果物が体系的に蓄積される構造
Skills        → 繰り返し作業の自動化（/コマンド）
Hooks         → 品質保証のための自動検証
```

---

## 基本の設計思想

> 「すべてのルールをCLAUDE.mdに入れないでください。Skillsで分散し、Hooksで強制してください。」

- **CLAUDE.md**：変わらない役割と原則
- **Skills**：繰り返される作業フロー
- **Hooks**：見落としてはいけない品質基準

---

## 用途別ワークスペースの例

### リサーチワークスペース

```
CLAUDE.md の要点:
  - 出典のない主張は禁止
  - すべての情報に出典グレードを表記（A〜E）
  - リサーチ結果は/reportsフォルダに自動保存

フォルダ構造:
  inbox/      → 参考資料の保管
  reports/    → 完成したリサーチ
  sources/    → 出典一覧

Skills:
  /research [テーマ]  → ウェブ検索 → 整理 → 保存
  /summary [ファイル]   → ドキュメント要約
```

### コンテンツ制作ワークスペース

```
CLAUDE.md の要点:
  - ブランドトーンの定義（例：親しみやすく実用的、専門用語は最小限）
  - ターゲット読者の定義
  - コンテンツ形式テンプレート

フォルダ構造:
  drafts/     → 草案
  published/  → 公開済み
  templates/  → 再利用形式

Skills:
  /blog [テーマ]       → ブログ記事の草案
  /newsletter [内容] → ニュースレター形式に変換
  /sns [内容]        → Instagram・X（Twitter）版
```

> 💡 リサーチワークスペースを構成する際、**deep-research**プラグインを活用すれば、別途Skillsを作成せずにすぐにマルチエージェントリサーチが可能です。コンテンツ作業には**docs-guide**で公式ドキュメントベースの正確な資料を確保できます。

### ビジネスワークフロー

```
CLAUDE.md の要点:
  - 会社/チーム情報、主要プロジェクトの現状
  - メールスタイル（フォーマル/カジュアルの度合い）
  - レポート形式テンプレート

Skills:
  /meeting [録音]    → 要約 + 決定事項 + アクションアイテム
  /report [テーマ]     → 週次レポート草案
  /email [状況]      → メール草案
```

## 実際の事例公開：アイディエーションワークスペース

CC101（このサイト）が生まれたワークスペースを公開します。

### CLAUDE.md の要点

```markdown
# アイディエーションワークスペース

## 役割
アイデアの発掘、検証、実行をサポートするAIパートナー

## 基本原則
- 日本語専用
- リサーチなしの主張は禁止（ハルシネーション防止）
- すべての出典にグレード表記（A〜E）
- 検証なしに「うまくいきそう」という発言は禁止
- アイデアは必ずドキュメント化（揮発防止）

## 検証方式
4つのペルソナCouncil:
- Visionary: 潜在力・ビジョン
- Pragmatist: 実現可能性・リソース
- Critic: リスク・弱点
- User Advocate: ユーザー価値

## フォルダ構造
10-inbox/      → 参考資料
20-sparks/     → アイデアの捕捉
30-research/   → リサーチ結果
40-foundry/    → 検証結果
50-blueprints/ → MVP設計
60-ventures/   → 実行中
70-playbook/   → 経験の蓄積
90-shelf/      → 保留アイデア
```

### 使用中のSkills

```
/spark [アイデア]   → アイデアの捕捉 + ICEスコア評価
/research [テーマ]    → 7段階ディープリサーチ
/validate           → 4ペルソナCouncil検証
/insight [内容]     → 教訓Playbook保存
```

### このワークスペースで生まれたもの

- CC101のアイデア捕捉と検証
- 3つのAIセクション構成ディスカッション（agent-council）
- 公式ドキュメント57件のリサーチ
- Next.jsサイトの開発とデプロイ
- その後の対話を通じた継続的な改善

## 自分だけのワークスペースを作る -- 4ステップ

### ステップ1：目的を定義する

```
自問してください：
□ どんな作業を繰り返しているか？
□ AIにどんな役割を任せたいか？
□ どんなルールを一貫して守ってほしいか？
□ 成果物がどこに蓄積されるのが良いか？
```

### ステップ2：CLAUDE.mdを作成する

最小構成テンプレート（コピーして始めてください）：

```markdown
# [ワークスペース名]

## 役割
[AIに与える役割と目的 -- 1-2文]

## 基本ルール
- [絶対に守ること 1]
- [絶対に守ること 2]
- [絶対にやってはいけないこと]

## フォルダ構造
- [フォルダ名]/  → [用途]
- [フォルダ名]/  → [用途]

## 主な作業方式
[繰り返されるワークフローの説明]
```

### ステップ3：フォルダ構造を設定する

番号プレフィックスを使うと順序が維持されます：

```bash
mkdir -p 01-inbox 02-work 03-output 04-archive
```

### ステップ4：繰り返し作業 → Skills化

同じ作業を3回以上するようになったらSkillにしましょう。
→ Skillsの作成方法は15セクションで解説しています

## MCPとの組み合わせ

ワークスペースにMCPを接続すると、さらに強力になります：

| MCP | 活用方法 |
|-----|------|
| Web Search | 自動リサーチ |
| GitHub | コード管理の自動化 |
| Slack | チームコミュニケーションの統合 |
| Google Sheets | データの自動集計 |

→ MCP接続は13セクションで解説しています

---

> ワークスペースは一度で完成するものではありません。使いながら不便な点が出てきたらCLAUDE.mdを修正し、繰り返し作業が見つかったらSkillを追加してください。時間が経つほど、自分に合った形に進化していきます。
