# 19. コスト管理 & 節約のヒント

> Claude Codeをよりスマートに、より安く使う方法を学びます。

---

## コスト構造の概要

Claude Codeの料金体系は大きく2種類あります。

| 区分 | 方式 | 特徴 |
|------|------|------|
| **Claude.ai Pro / Max** | 定額制サブスクリプション | 月額固定料金、API費用は別途不要 |
| **Anthropic API** | 従量制 | 使用したトークン数に応じて料金が発生 |

**Claude.ai Pro/Maxサブスクリプション利用者**は、サブスクリプション料金にClaude Codeの利用が含まれているため、API費用は別途請求されません。`/cost`コマンドが表示する数値はAPIユーザー向けのものなので、サブスクリプション利用者は参考程度にご覧ください。

**APIユーザー**は使用量に応じて料金が発生します。公式ドキュメントによると、平均して開発者1人あたり1日約$6、90%のユーザーは1日$12以下のコストとなっています。

---

## トークンとは？

**トークン(token)**はClaudeがテキストを処理する単位です。

- 英語の場合：約1単語 = 1〜2トークン
- 日本語の場合：約1〜2文字 = 1トークン
- 大まかに：**英語4文字 ≈ 1トークン**

簡単に言えば、Claudeにファイルを読ませたり、質問したり、回答を受け取るたびにトークンが消費されます。コンテキスト（会話の文脈）が長くなるほど、より多くのトークンが使用されます。

---

## コストが高くなる状況

### 1. 長いファイルを繰り返し読む場合

Claudeが同じファイルを何度も読むと、そのたびにトークンが消費されます。10,000行のログファイルをまるごと読ませると、1回で数万トークンが使用されます。

### 2. コンテキストを長時間維持する場合

会話が長くなるほど、以前の内容をすべてコンテキストに含める必要があります。関連のない作業を同じセッションで続けると、不要な情報が蓄積されコストが増加します。

### 3. 高性能モデルを使用する場合

OpusモデルはSonnetより強力ですが、トークンあたりのコストも高くなります。簡単な作業にOpusを使うのは非効率的です。

### 4. 自動化（CI/CD）で無制限に実行する場合

GitHub Actionsなどで自動化すると、人の介入なしに大量のトークンが消費される可能性があります。

## 節約のヒント5選

### ヒント1：`/compact`でコンテキストを整理する

会話が長くなったとき、`/compact`コマンドを実行すると、以前の内容を要約してコンテキストサイズを削減してくれます。

```
/compact コード変更とテスト結果を中心に要約して
```

これにより重要な情報は維持しながら、不要な会話内容が圧縮されます。

CLAUDE.mdに圧縮方法をあらかじめ指定することもできます：

```markdown
# Compact instructions

When you are using compact, please focus on test output and code changes
```

> **ヒント**：`/clear`を使うとコンテキストを完全にリセットできます。まったく別の作業を始めるときに便利です。

---

### ヒント2：不要なファイル参照を避ける

曖昧なリクエストは、Claudeに不必要に多くのファイルを読ませてしまいます。

| 非効率的なリクエスト | 効率的なリクエスト |
|----------------|--------------|
| 「このコードベースを改善して」 | 「auth.tsのlogin関数に入力バリデーションを追加して」 |
| 「バグを探して」 | 「セッション期限切れ後にログインが失敗する問題をsrc/auth/で確認して」 |

具体的なリクエストほど、Claudeが必要なファイルだけを読むためトークン消費が減ります。

---

### ヒント3：Haikuモデルを活用する

すべての作業に高性能モデルを使う必要はありません。

| モデル | 用途 | コスト |
|------|------|------|
| **Haiku** | 簡単な質問、ファイル要約、単純な作業 | 低い |
| **Sonnet** | 一般的なコーディング作業（デフォルト推奨） | 中程度 |
| **Opus** | 複雑なアーキテクチャ設計、難しい推論 | 高い |

モデルの切り替えはセッション中でも可能です：

```
/model haiku
```

または起動時に指定：

```bash
claude --model haiku
```

**`opusplan`モード**：計画段階ではOpus、実行段階ではSonnetに自動で切り替わるハイブリッドモードです。

```
/model opusplan
```

---

### ヒント4：Fast Modeを理解して活用する

`/fast`コマンドでFast Modeをオンにすると、Opus 4.6が約2.5倍速く応答します。

```
/fast
```

ただし、Fast Modeは**トークンあたりのコストが高くなります**。速度が重要な場面（素早い反復作業、リアルタイムデバッグ）でのみ使用し、長時間の自律作業ではオフにするのがおすすめです。

> **注意**：Fast Modeはサブスクリプションプランの基本使用量には含まれず、追加使用（extra usage）として請求されます。

---

### ヒント5：`/cost`で定期的に確認する

APIユーザーであれば、現在のセッションのトークン使用量を確認できます：

```
/cost
```

出力例：
```
Total cost:            $0.55
Total duration (API):  6m 19.7s
Total duration (wall): 6h 33m 10.2s
Total code changes:    0 lines added, 0 lines removed
```

サブスクリプション利用者は`/stats`で使用パターンを確認できます。

ステータスバー（status line）を設定すると、コンテキスト使用量をリアルタイムで確認することもできます。

---

### ヒント6：CLAUDE.mdをダイエットする

CLAUDE.mdはClaudeが**毎回の応答ごとに**読むファイルです。長ければ長いほど、毎回より多くのトークンを消費します。

**実際の効果**：過度に記述されたCLAUDE.md（約19,000トークン）を要点だけ残したバージョン（約9,000トークン）に整理すると、同じ作業にかかるコストがほぼ半分に減ります。

**ダイエットの基準**

```
削除してよいもの：
- 長い背景説明（「このプロジェクトは2023年から...」）
- 繰り返される当たり前の内容（「常にベストを尽くしてください」）
- 複数のサンプルコードブロック（1つだけ残すか削除）
- 曖昧なルール（「良いコードを書いてください」）

必ず残すもの：
- 禁止事項（絶対にやってはいけないこと）
- プロジェクトの技術スタック（1〜3行の要約）
- よく参照するファイルパス
- 言語/フォーマットのルール（具体的なもののみ）
```

ほとんどのCLAUDE.mdは整理すると50%以上削減できます。

---

### ヒント7：`.claudeignore`で不要なファイルを除外する

`node_modules`、`.git`、ビルド成果物など、Claudeが読む必要のないファイルを明示的に除外すると、Claudeが誤って大容量ファイルを読むことを防げます。

`.gitignore`と同じ構文で、プロジェクトルートに`.claudeignore`ファイルを作成してください：

```
# .claudeignore
node_modules/
.next/
dist/
build/
coverage/
*.log
*.lock
.git/
```

**特に効果的な場面**
- モノレポのようにファイル数が多いプロジェクト
- 「このプロジェクト全体を分析して」というリクエストを頻繁に行う場合
- GitHub Actionsなどで Claudeを繰り返し実行する場合

## モデル別コスト比較（概要）

| モデル | 特徴 | API基準 |
|------|------|----------|
| Haiku | 高速で低コスト、簡単な作業 | 最も安い |
| Sonnet | バランスの取れた性能、一般的なコーディング作業 | 中程度 |
| Opus | 最高性能、複雑な推論 | 最も高い |
| Opus (Fast Mode) | Opus速度2.5倍、トークンコスト高い | Opusより高い |

> 正確なトークン価格は[Anthropic料金ページ](https://www.anthropic.com/pricing)をご参照ください。

---

## Claude.ai Maxプランのメリット

Claude.ai ProまたはMaxサブスクリプション利用者は、**API従量制の費用なしで**Claude Codeを使用できます。

- 月額料金にClaude Codeの使用が含まれる
- 追加のAPIキーが不要
- コストを気にせず様々な実験が可能
- Maxプランの場合、より高い使用量上限

> **ただし、Fast Modeと1Mコンテキスト超過分は追加使用（extra usage）として請求される場合があります。**

---

## チームのコスト管理

APIをチーム単位で使用する場合：

- **Workspaceの支出上限を設定**：[Anthropic Console](https://platform.claude.com)でチーム全体の支出上限を設定できます。
- **使用量ダッシュボード**：Consoleでメンバーごとのコストと使用量を確認できます。
- **エージェントチームに注意**：Agent Teams（複数のClaudeインスタンスを同時実行）機能は、標準セッションの約7倍のトークンを消費します。

## まとめ：コスト節約チェックリスト

```
✅ 作業が終わったら/clearでコンテキストをリセット
✅ 関連のない作業は新しいセッションで開始
✅ 具体的なファイル名と関数名を明示
✅ 簡単な作業にはHaikuモデルを使用
✅ Fast Modeは必要なときだけオンにする
✅ /costまたは/statsで定期的に確認
✅ CLAUDE.mdにcompact指示を設定
✅ CLAUDE.mdは重要なルールだけ残して定期的にダイエット
✅ .claudeignoreでnode_modules、ビルド成果物を除外
```
