# 09. コマンド一覧

> Claude Codeを使う際に知っておくべきコマンドを一覧で整理しました。

---

## コマンドの3つの種類

Claude Codeのコマンドは大きく3つに分かれます。

| 種類 | どこで入力？ | 例 |
|------|-------------|------|
| **CLIコマンド** | ターミナル（Claude Codeの外） | `claude`, `claude --version` |
| **スラッシュコマンド** | 会話中（Claude Codeの中） | `/help`, `/compact` |
| **キーボードショートカット** | 会話中 | `Ctrl+C`, `Shift+Enter` |

---

## CLIコマンド（ターミナルで入力）

Claude Codeを起動・制御するコマンドです。ターミナルで入力します。

### 基本実行コマンド

| コマンド | 説明 | 例 |
|--------|------|------|
| `claude` | 対話モードを開始 | `claude` |
| `claude "質問"` | 質問しながら開始 | `claude "このプロジェクトを説明して"` |
| `claude -p "質問"` | 回答だけ出力して終了（自動化に便利） | `claude -p "この関数を説明して"` |
| `claude -c` | 最新の会話を引き継いで開始 | `claude -c` |
| `claude --version` または `claude -v` | インストール済みバージョンの確認 | `claude -v` |
| `claude --help` | ヘルプを表示 | `claude --help` |
| `claude update` | 最新バージョンにアップデート | `claude update` |

### モデルを指定して実行

```bash
# 特定のモデルで開始
claude --model claude-opus-4-5

# または短縮形で
claude --model opus
claude --model sonnet
claude --model haiku
```

### 便利な起動オプション

```bash
# Plan Modeで開始（ファイル修正なしで計画のみ）
claude --permission-mode plan

# セッション名で引き継いで開始
claude --resume セッション名

# 以前の会話一覧から選んで引き継いで開始
claude --resume
```

> **ヒント**：最初はただ `claude` と入力して始めれば大丈夫です。残りは必要になったときに覚えても遅くありません。

## スラッシュコマンド（会話中に入力）

Claude Codeとの会話中に `/` で始まるコマンドを入力すると、特別な機能を使用できます。

### よく使うスラッシュコマンド

| コマンド | 説明 |
|--------|------|
| `/help` | 使用可能なコマンドの全リストを表示 |
| `/compact` | 会話内容を圧縮（長いセッションでは必須！） |
| `/clear` | 会話を初期化（完全にやり直し） |
| `/cost` | 現在のセッションで使用したトークンコストを確認 |
| `/model` | 使用中のAIモデルを変更 |
| `/permissions` | 権限設定の確認と管理 |
| `/memory` | CLAUDE.mdメモリファイルを編集 |
| `/quit` または `/exit` | Claude Codeを終了 |
| `/plan` | Plan Modeに切り替え |
| `/init` | プロジェクトにCLAUDE.mdファイルを生成 |
| `/doctor` | インストール状態を診断 |
| `/stats` | 使用量統計を表示（サブスクリプションユーザー用） |
| `/vim` | Vim編集モードのオン/オフ |
| `/theme` | カラーテーマの変更 |
| `/clone` | 現在の会話を複製して独立した分岐を作成（実験に便利） |
| `/half-clone` | 圧縮要約で新しい会話分岐（コンテキスト節約） |
| `/context` | コンテキストウィンドウのトークン使用量を可視化 |
| `/rename [名前]` | セッションに名前を付ける（`--resume 名前`で再開） |
| `/bug` | Anthropicにバグレポートを直接送信 |

### スラッシュコマンドの使い方

```
> /compact
→ 会話内容を要約してコンテキストを削減します

> /compact コード変更点を中心に整理して
→ 圧縮の方向を直接指定できます

> /model
→ モデル選択画面が表示されます
```

> **ヒント**：会話画面で `/` だけ入力すると、使用可能なコマンド一覧がオートコンプリートで表示されます。

## 会話の分岐と実験の戦略

会話の途中で方向を変えたり、リスクのある実験を安全に試したいときに活用できる機能です。

### `/clone` — 元の会話を保持したまま実験する

現在の会話全体を複製して、独立した分岐を作ります。複雑なリファクタリングや構造的な変更を試す前に現在の状態をコピーしておけば、失敗しても元の会話に戻れます。

```
> /clone
→ 現在の会話状態をそのまま新しい分岐として開始
→ 元の会話には影響なし
```

**こんなときに使ってみてください**
- 複数の実装方法のうち1つをまず試してみたいとき
- 「こう直したらどうなるかな」と思うが失敗が心配なとき
- 同じ問題を異なるアプローチで比較してみたいとき

### `/half-clone` — より軽量に新たなスタート

現在の会話をAIが圧縮要約した上で、新しい会話を開始します。`/compact`は既存の会話内で圧縮するのに対し、`/half-clone`は圧縮した要約をもとに完全に新しい会話を開きます。

```
> /half-clone
→ 核心情報だけを含む要約で新しい会話を開始
→ トークン使用量が大幅に減少
```

会話が長くなりすぎて`/compact`でも足りないときに便利です。

### `/context` — コンテキスト使用量の確認

```
> /context
→ [■■■■■■■□□□] 70% (143,000 / 200,000 tokens)
```

現在の会話がコンテキストウィンドウの何%を占めているか視覚的に表示します。80%以上になったら`/compact`や`/half-clone`を検討してください。

### `/rename` — セッションに名前を付ける

重要な作業セッションに名前を付けておくと、後から簡単に再開できます。

```
> /rename 競合調査-2月

# 後でターミナルから引き継いで開始
$ claude --resume 競合調査-2月
```

## キーボードショートカット

会話中にキー操作で素早く制御できます。

### 主要ショートカット

| ショートカット | 説明 |
|--------|------|
| `Ctrl+C` | 現在の作業をキャンセル（Claudeが何かしている途中で止めたいとき） |
| `Ctrl+D` | Claude Codeを終了 |
| `Ctrl+L` | ターミナル画面をクリア（会話内容は保持） |
| `Ctrl+R` | 以前入力したコマンドを検索 |
| `Ctrl+G` | 現在入力中の内容を外部テキストエディタで編集 |
| `Ctrl+T` | タスクリストの表示/非表示 |

### 入力関連ショートカット

| ショートカット | 説明 |
|--------|------|
| `Shift+Enter` | 改行（長い指示を複数行で入力するとき） |
| `Option+Enter`（macOS） | 改行（macOSデフォルト） |
| `\` + `Enter` | 改行（すべてのターミナルで動作） |
| `↑` / `↓` 矢印キー | 前/次のコマンドを参照 |
| `Esc` + `Esc`（2回） | 会話を巻き戻し（前の状態に復元） |

### モード切り替えショートカット

| ショートカット | 説明 |
|--------|------|
| `Shift+Tab` | Plan Mode ↔ 通常モード切り替え |
| `Option+P`（macOS） / `Alt+P` | モデル変更 |
| `Option+T`（macOS） / `Alt+T` | 拡張思考モードのオン/オフ |
| `Ctrl+B` | 現在の作業をバックグラウンドに保持し、新しい会話ウィンドウを開く |

## 初心者がよく使うコマンドTOP 5

始めたばかりなら、この5つだけ覚えれば十分です。

### 1. `/compact` — 最も重要なコマンド

```
> /compact
```

会話が長くなると、Claudeが遅くなったり古い内容を忘れたりします。このとき `/compact` と入力すると会話を圧縮して速度を回復します。**長い作業中は定期的に使う習慣をつけましょう。**

### 2. `Ctrl+C` — 止めたいとき

Claudeが見当違いの方向に進んでいるとき、または誤って危険なコマンドを許可してしまったときに即座に停止します。

### 3. `/clear` — 最初からやり直すとき

```
> /clear
```

まったく別のトピックに移るときに使います。以前の会話内容はすべて消去されます。

### 4. `Shift+Enter` — 長い指示の入力

1行で書ききれない長いリクエストは `Shift+Enter` で改行しながら入力してください。

```
（Shift+Enterで複数行入力の例）
ログインページを作って。
- メールアドレスとパスワードのフィールド
- パスワードの表示/非表示ボタン
- ログインボタン（ローディング状態付き）
```

### 5. `/cost` — コスト確認

```
> /cost
```

現在のセッションでどれくらい使ったか確認します。（APIキーユーザー向け、サブスクリプションユーザーは `/stats` を使用）

<details>
<summary>🖥️ 開発者向け：ターミナルAlias設定（ショートカットコマンド）</summary>

毎回長いコマンドを入力する代わりに、短いaliasを登録しておくとずっと便利です。

### macOS / Linux（zsh）

`~/.zshrc` ファイルに追加：

```bash
alias cc='claude'
alias ccd='claude --dangerously-skip-permissions'
alias ccr='claude --resume --dangerously-skip-permissions'
```

追加後に適用：

```bash
source ~/.zshrc
```

### Windows（PowerShell）

`$PROFILE` ファイルに追加：

```powershell
function cc { claude @args }
function ccd { claude --dangerously-skip-permissions @args }
function ccr { claude --resume --dangerously-skip-permissions @args }
```

PowerShellを再起動すると適用されます。

### 各Aliasの説明

| コマンド | 意味 | 使用タイミング |
|--------|------|----------|
| `cc` | 基本実行 | 常時 |
| `ccd` | 権限確認を自動承認 | 信頼できるプロジェクト、素早い作業 |
| `ccr` | 前回のセッション再開 + 権限自動承認 | 昨日の作業の続き |

> ⚠️ **注意**：`--dangerously-skip-permissions`はファイル修正、コマンド実行などすべての権限確認を自動的に承認します。自分で作ったプロジェクト、信頼できる環境でのみ使用してください。

</details>
