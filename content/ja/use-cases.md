# 02. Claude Codeでできること

> コードを一行も書かずにこのサイト（CC101）を作りました。Claude Codeはコーディング専用ツールではありません。

---

## まず: Web UI（Claude.ai）とは何が違うの？

| | Claude.ai（Web） | Claude Code（ターミナル） |
|--|---------------|---------------------|
| ファイルアクセス | 手動で添付のみ | フォルダ全体の読み込み・編集 |
| 作業範囲 | 1回の会話 | 数十個のファイルを同時処理 |
| 外部実行 | 不可 | ターミナルコマンドを直接実行 |
| 繰り返し自動化 | 毎回手動 | Skills・Hooksで自動化 |
| 記憶 | 会話内のみ | CLAUDE.mdで永続的に記憶 |

一言で言えば: **Claude.aiが会話相手なら、Claude Codeは自分のパソコンで働くAIの同僚**です。

## できること

### 開発作業
コード作成、バグ修正、リファクタリング、テスト、コードレビュー
→ このガイドの多くのセクションで取り上げます

### リサーチ＆分析
- 複数のWebページを同時に読み込んでMarkdownで整理
- 競合分析、市場調査、技術トレンドの把握
- PDF・ドキュメント数十件の一括要約

> 💡 **deep-research**プラグインを使用すると、マルチエージェントが複数のソースを並列で調査し、総合レポートを自動生成します。

### ドキュメント＆コンテンツ制作
- ブログ記事、ニュースレター、SNSコンテンツの下書き
- 提案書、レポート、プレゼン資料
- 日本語 ↔ 英語の翻訳後、ファイルとして自動保存

> 💡 **docs-guide**プラグインで公式ドキュメントに基づいた正確なコンテンツを作成できます。

### データ処理
- CSV、Excel分析および変換 — **Pythonを知らなくても大丈夫です**
- 数百個のファイルの一括処理（名前変更、形式変換、フォルダ整理）
- JSON、XMLデータのパースおよび加工

### 学習サポート
- 講義資料、書籍の内容を構造化して整理
- クイズ、フラッシュカードの自動生成
- 概念説明 + 例題 + 練習問題

### ビジネスワークフロー
- ミーティング録音テキスト → 要約 + 決定事項 + 担当者別アクションアイテム
- 週次・月次レポートの自動化
- メール下書き、顧客対応テンプレート

### パーソナルAIワークスペース
CLAUDE.mdで役割とルールを定義すれば、単なるAIチャットではなく**自分だけのAIエージェントシステム**になります。
→ Advanced 17セクションで詳しく取り上げます

### バイブコーディング
コーディングを知らなくてもAIへの指示だけでソフトウェアを作る**バイブコーディング**の中核ツールです。
- 企画書の自動生成（show-me-the-prdプラグイン）
- Gitバージョン管理の自動化（バルダ Git先生プラグイン）
- AI使用パターン分析＆コーチング（バ先生プラグイン）
→ **バイブコーディングとは？** セクションで詳しく取り上げます

## 実際の事例: CC101はどうやって作られたか

> **コードを一行も自分で書いていません**

**企画**
「Codex 101のような日本語版Claude Codeガイドを作りたい」
→ アイデアワークスペースで3つのAI（Claude・Codex・Gemini）がセクション構成、UX、技術スタックを議論
→ 20セクションの構成を決定

**コンテンツ制作**
公式ドキュメント57件を収集 → Claudeが要点を抽出 → 韓国語・英語のMarkdownとして構造化

**開発**
Next.jsサイト全体のコード → Claude Codeが作成
ダーク/ライトモード、韓英切り替え、モバイル対応 → すべて会話で処理

**デプロイ**
Vercelデプロイ、Cloudflare DNS設定、ドメイン接続 → Claude Codeが直接処理

**運用＆改善**
初稿以降、ユーザーが内容を確認し「この部分を追加して」「これは別の書き方にして」と
会話を通じて継続的に改善中 — **今この会話もCC101アップデートセッションです**

## 非開発者の実際の使用例

```
状況: 毎週のチームミーティング後に議事録を整理する必要がある

従来の方法:
  ミーティング終了 → メモ整理30分 → メール作成10分 → 共有

Claude Codeの方法:
  ミーティング録音テキストを貼り付け
  → 「要約、決定事項、担当者別アクションアイテムを整理して」
  → Markdownファイルとして保存 + メール下書きを自動生成
  所要時間: 3分
```

## 今すぐ体験

インストールが完了していれば、今すぐ始められます:

```
1. ドキュメント要約
   フォルダにPDFやテキストファイルを入れて:
   「このドキュメントの要点を3行で要約して」

2. メール下書き
   「次の状況でメールの下書きを作成して: [状況の説明]」

3. データ処理
   CSVファイルをフォルダに入れて:
   「このデータを[カラム名]基準でソートして要約して」
```

コーディング経験がまったくなくても大丈夫です。
