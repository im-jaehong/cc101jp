# 10. 初心者ワークフロー

> 初めてClaude Codeを使うとき、どのように始めて、どのように作業すればよいか、ステップごとにご案内します。

---

## 初回セッションの流れ（Step-by-Step）

初めての方は、この順番に沿って試してみてください。

### Step 1. プロジェクトフォルダに移動

```bash
cd myproject
```

Claude Codeは現在のフォルダを基準にファイルを読み書きします。必ず作業するプロジェクトフォルダに先に移動してください。

> フォルダがない場合は `mkdir myproject && cd myproject` で作成しても構いません。

### Step 2. Claude Codeを起動

```bash
claude
```

ターミナルで `claude` と入力してEnterを押すと、対話モードが開始されます。

### Step 3. プロジェクトを把握させる（強くおすすめ）

最初に始めるとき、Claudeにプロジェクトを把握させるか、CLAUDE.mdファイルを作成するよう依頼しましょう。

```
> このプロジェクト全体を把握してCLAUDE.mdファイルを作ってください
```

既存のプロジェクトであれば：

```
> このコードベースの構造を説明してください
```

**CLAUDE.mdが重要な理由**：このファイルにプロジェクトのルール、技術スタック、注意事項を書いておけば、Claudeが毎回質問しなくても済みます。チームがある場合はgitにコミットして共有しましょう。

### Step 4. 作業を指示

やりたい作業を自然言語で指示します。

```
> ログイン機能を作ってください
> このコードのバグを見つけてください
> README.mdを日本語に翻訳してください
```

### Step 5. 結果確認とフィードバック

Claudeがファイルを修正すると、変更内容を表示します。気に入らなければすぐにフィードバックしましょう。

```
> いいですね、ただエラーメッセージは日本語にしてください
> この部分は別の方法でやってください — [説明]
```

### Step 6. セッション終了

```
> /quit
```

または `Ctrl+D` を押すと終了します。

## 効果的なプロンプトの書き方

### 具体的に指示するほど良い結果が出ます

| 悪い例（曖昧） | 良い例（具体的） |
|----------------|----------------|
| 「コードを直して」 | 「login.jsの42行目で発生するTypeErrorを修正して」 |
| 「機能を追加して」 | 「ユーザーがパスワードを忘れたとき、メールでリセットリンクを送る機能を追加して」 |
| 「もっと良くして」 | 「この関数のパフォーマンスを改善して。現在1000件のデータで3秒かかるが、1秒以内に短縮したい」 |
| 「テストを作って」 | 「auth.jsのlogin関数の単体テストを作って。成功/失敗/メール形式エラーのケースを含めて」 |

### 一度に一つずつリクエストする

複数の要件を一度にまとめて依頼すると、Claudeが途中で方向を見失うことがあります。

```
# 非推奨
> ログインを作って、会員登録も追加して、パスワード再設定も作って、
  メール認証も入れて、ソーシャルログインも追加して

# 推奨
> まずメール/パスワードのログイン機能だけ作ってください
（完了後）
> 次に会員登録機能を追加してください
```

### 出力形式を明示する

```
> このコードを説明してください         → 説明文として回答
> このコードをファイルに保存してください   → ファイルとして保存
> このコードを改善して上書きしてください   → ファイルを直接修正
> バグ一覧を表で整理してください         → 表形式で回答
```

### @記号でファイルを指定する

`@` の後にファイルパスを付けると、Claudeがそのファイルを直接読み込んで作業します。「このファイルを読んで...」と説明する必要なく、ファイルを会話に直接渡すことができます。

```
> @議事録.txt からアクションアイテムだけ抽出してください
> @契約書.pdf の重要条項だけ整理してください
> @データ.csv を分析してインサイト3つ抽出してください
> @src/auth/login.js このファイルのバグを見つけてください
> @README.md を見てプロジェクト構造を把握してください
```

フォルダ全体を渡すこともできます：

```
> @docs/ このフォルダの文書をすべて読んで目次を作ってください
> @reports/2025/ 月別レポートを比較してトレンド分析をしてください
```

> **ヒント**: 入力欄で `@` を入力するとファイル一覧がオートコンプリートで表示されます。日本語ファイル名でも問題ありません。

## Plan Modeの活用ヒント

**Plan Modeとは？** Claudeがファイルを修正せず、計画だけを立てるモードです。リスクのある作業を行う前に、まず計画を確認できます。

### Plan Modeの有効化

```
Shift+Tabを2回押す
または
> /plan
```

### どんなときにPlan Modeを使うか

- 複数のファイルを同時に修正する必要があるとき
- データベースマイグレーションのように元に戻しにくい作業
- 初めて見るコードベースを修正する前
- アーキテクチャ変更のような大きな作業

> 💡 プロジェクトの初期企画段階では、**show-me-the-prd**プラグインでインタビューベースの企画書を自動生成できます。

### Plan Modeの使用例

```bash
# Plan Modeで開始
claude --permission-mode plan

# または会話中に切り替え
> Shift+Tab（2回）

# 計画策定を依頼
> 認証システムをJWTからOAuth2にマイグレーションする計画を立ててください

# Claudeが計画を表示したら確認
# 問題なければ通常モードに切り替えて実行
```

## 作業前のgit commit習慣

Claude Codeがファイルを修正する前に、現在の状態をgitに保存しておけば、ミスをしても簡単に元に戻せます。

```bash
git add .
git commit -m "Claude Code作業前のバックアップ"
```

> 💡 Gitコマンドに慣れていない場合は、Claude Codeに「gitで現在の状態を保存して」と依頼するだけでも対応できます。

その後、Claudeが修正した内容が気に入らなければ：

```bash
git diff          # 変更内容を確認
git checkout .    # すべての変更を取り消し（注意！）
```

またはClaude Code内で `Esc` + `Esc` を押して前の状態に巻き戻すこともできます。

## ⚠️ 絶対にやってはいけないこと

### 機密情報の入力禁止

```
# 絶対禁止
> 私のAWSアクセスキーはAKIAXXXXXXXXで...
> DBパスワードはmypassword123で...
> 私のSSH秘密鍵は-----BEGIN RSA...
```

APIキー、パスワード、個人証明書などは、絶対に入力欄に直接入力しないでください。環境変数や `.env` ファイルを使用してください。

### 権限リクエストの無分別な許可禁止

Claudeが `sudo` コマンドやシステムファイルの修正権限を要求するとき、まず理由を確認してください。

```
# Claudeがこのような権限を要求した場合、まず理由を確認
> なぜsudoが必要ですか？他の方法はありませんか？
```

### `--dangerously-skip-permissions` の使用に注意

このオプションはすべての権限確認をスキップします。隔離されたテスト環境（コンテナなど）でなければ使用しないでください。

### 重要な作業前にバックアップなしで進行しない

大規模なリファクタリングやデータベースマイグレーションの前には、必ずgit commitやバックアップを先に行ってください。

## セッション管理のヒント

### 長い作業には名前を付けて保存

```
> /rename auth-refactor
```

後で続きをするとき：

```bash
claude --resume auth-refactor
```

### コンテキストが長くなったら圧縮

作業が長くなるとClaudeの応答が遅くなります。定期的に：

```
> /compact
```

### 関連のない新しい作業は初期化してから開始

```
> /clear
```
