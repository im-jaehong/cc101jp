# 07. コアコンセプトを理解する

> Claude Codeがどのように動作するかを知ると、より上手に使えるようになります。

---

## Claude Codeはどのように動作するのか？

Claude Codeは単なるチャットボットではありません。指示を出すと、自ら計画を立て、ファイルを読み、コードを修正し、結果を確認する**エージェント（Agent）**です。

このプロセスを**エージェントループ（Agentic Loop）**と呼びます。

```
あなたの指示
     ↓
1. 状況把握（Context収集）
   - ファイル読み込み、コード検索、現在の状態把握
     ↓
2. 行動（Action）
   - ファイル修正、コマンド実行、Web検索
     ↓
3. 結果確認（Verify）
   - テスト実行、エラー確認、再修正
     ↓
作業完了（または1に戻って繰り返し）
```

例えば「reports/ フォルダのドキュメントを要約して」と言うと、Claude Codeは：
1. フォルダ内のファイル一覧を確認
2. 各ドキュメントを読んで内容を把握
3. 要約・整理を作成
4. 結果を確認し、不足があれば補完

この全プロセスを**自動的に**実行します。

<details>
<summary>🖥️ 開発者向けの例（バグ修正）</summary>

「テストが失敗するバグを直して」と言うと：
1. テストを実行してどんなエラーか確認
2. 関連するソースファイルを探して読む
3. コードを修正
4. 再度テストを実行して通るか確認

</details>

## 5つのコアコンセプト

### 1. Context（コンテキスト） — Claudeの作業記憶

**コンテキスト**とは、Claude Codeが現在記憶しているすべての内容です。会話内容、読んだファイル、実行したコマンドの結果、CLAUDE.mdの指示事項など、すべてがコンテキストに含まれます。

コンテキストには**トークンの上限**があります。会話が長くなったり、大きなファイルを多く読んだりすると、コンテキストがいっぱいになることがあります。その場合、Claude Codeが自動的に古い内容を整理します。

```
# コンテキストの状況確認
/context

# 会話の圧縮（古い内容を要約）
/compact

# 特定のテーマに集中して圧縮
/compact API変更点に集中して圧縮して
```

> **ヒント**：頻繁に参照する重要なルールや情報はCLAUDE.mdに書いておくと、毎回コンテキストを消費せずにClaudeが記憶できます。

---

### 2. Tools（ツール） — Claudeが実際にできること

Claude Codeは会話するだけではありません。実際に様々な**ツール（Tools）**を使って作業を実行します。

| ツールの種類 | Claudeができること |
|----------|----------------------|
| **ファイル操作** | ファイル読み込み、コード編集、新規ファイル作成、ファイル名変更 |
| **検索** | ファイルパターンで検索、正規表現で内容検索、コードベース探索 |
| **実行** | シェルコマンド実行、サーバー起動、テスト実行、Git操作 |
| **Web** | Web検索、公式ドキュメント参照、エラーメッセージ検索 |
| **コード分析** | 編集後の型エラー検出、定義へのジャンプ、参照の検索 |

---

### 3. Permissions（権限） — 何を許可するか設定する

Claude Codeは強力なツールであるため、**何を許可するかを直接制御**できます。

ファイルの読み込みはデフォルトで許可されていますが、ファイルの修正やシェルコマンドの実行はユーザーの確認を求めます。

権限ルールは大きく3種類です：

| ルール | 説明 |
|------|------|
| **Allow（許可）** | 確認なしで自動的に実行 |
| **Ask（確認要求）** | 実行前に承認を求める |
| **Deny（拒否）** | 絶対に実行しない |

```
# 権限設定の確認と管理
/permissions
```

> **優先順位**：Deny > Ask > Allow の順に適用されます。拒否ルールが常に最優先です。

---

### 4. Memory（メモリ） — CLAUDE.mdでプロジェクトを記憶させる

Claude Codeはセッションが終わると会話内容を記憶しません。しかし**CLAUDE.md**ファイルを作っておくと、毎セッション自動的に読み込んでプロジェクトのコンテキストを維持します。

CLAUDE.mdはClaudeに渡す**指示書**だと考えてください。

| ファイルの場所 | 適用範囲 |
|----------|---------|
| `./CLAUDE.md` | 現在のプロジェクトのみ適用（チーム共有可能） |
| `~/.claude/CLAUDE.md` | すべてのプロジェクトに適用（グローバル設定） |
| `./CLAUDE.local.md` | 自分だけのプロジェクト別設定（gitにアップされない） |

> CLAUDE.mdの詳細については**[08. CLAUDE.md 詳細ガイド](./06-claude-md.md)**で解説します。

---

### 5. Session（セッション） — 会話の始まりと終わり

**セッション**とは、Claude Codeを起動してから終了するまでの1つの作業単位です。

- 新しいセッションを開始すると、以前の会話内容はなくなります（CLAUDE.mdは維持）
- セッション中に行った作業は保存されます

```bash
# 最新のセッションを引き継いで開始
claude --continue

# 以前のセッション一覧から選んで再開
claude --resume

# 別の方向を試すためにセッションを複製（フォーク）
claude --continue --fork-session
```

## 「なぜ時々止まって確認を求めるのですか？」

Claude Codeが作業中に止まって確認を求めるのは、**意図された安全装置**です。

ファイルを修正したりコマンドを実行する前に、ユーザーが意図した操作かどうか確認しているのです。特に元に戻しにくい操作（ファイル削除、デプロイなど）ほど重要です。

---

## 初心者が知っておくべき3つの権限モード

`Shift+Tab` を押すと権限モードを切り替えることができます。

| モード | 動作方式 | いつ使うか |
|------|----------|------------|
| **Default（デフォルトモード）** | ファイル修正とシェルコマンド実行前に確認を要求 | 初めて使うときや重要な作業をするとき |
| **Auto-Accept（自動承認）** | ファイル修正は自動許可、シェルコマンドは引き続き確認要求 | 素早くコーディング作業を進めるとき |
| **Plan Mode（計画モード）** | ファイルを読んで分析するだけで、何も修正しない | まず計画を確認したいとき |

### モード切り替え方法

```
# ターミナルでShift+Tabを押すとモードが順番に切り替わります
# Default → Auto-Accept → Plan Mode → Default → ...

# またはPlan Modeに直接切り替え
/plan
```

### モード比較の例

```
「フォルダ内のドキュメントで『旧バージョンの文言』を『新しい文言』に一括置換して」と言った場合：

Defaultモード：
  → 各ファイルの修正前に「このファイルを修正してもいいですか？」と確認

Auto-Acceptモード：
  → ファイルは自動で修正、コマンド実行は確認要求

Plan Mode：
  → 「どのファイルのどの部分を変更する予定です」と表示するだけで、実際の修正はしない
```

<details>
<summary>🖥️ 開発者向けの例</summary>

```
「すべてのJavaScriptファイルからconsole.logを削除して」と言った場合：

Defaultモード：
  → 各ファイルの修正前に「このファイルを修正してもいいですか？」と確認

Auto-Acceptモード：
  → ファイルは自動で修正、コマンド（例：git commit）は確認要求

Plan Mode：
  → 「どのファイルの何行目を修正する予定です」と表示するだけで、実際の修正はしない
```

</details>

> **おすすめ**：最初は**Defaultモード**で始めて、Claude Codeが何をしているか把握してから、慣れてきたら**Auto-Acceptモード**に切り替えましょう。

## チェックポイント：ミスしても元に戻せます

Claude Codeはファイルを修正する前に**自動でスナップショット**を保存します。

誤って修正してしまった場合：
- `Esc` キーを2回連続で押して前の状態に戻す
- またはClaudeに「今やったことを取り消して」とリクエスト
