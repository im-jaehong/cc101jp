# 13. よくあるミスと解決法

> 初心者がClaude Codeを使う際によく遭遇する問題をQ&A形式でまとめました。

---

## Q1. `claude` コマンドが見つからないと表示される（command not found）

### 症状

```bash
$ claude
zsh: command not found: claude
```

### 原因と解決法

インストールはされたが、ターミナルが `claude` コマンドの場所を認識していない状態です。

**方法1：ネイティブインストールを使用（最も確実）**

```bash
# macOS / Linux / WSL
curl -fsSL https://claude.ai/install.sh | bash
```

インストール後、ターミナルを完全に閉じて新しく開いてください。

**方法2：PATHの確認**

```bash
# claudeがどこにインストールされたか確認
which claude

# PATHの確認
echo $PATH
```

`~/.local/bin` がPATHに含まれていない場合、シェル設定ファイル（`.zshrc` または `.bashrc`）に追加：

```bash
export PATH="$HOME/.local/bin:$PATH"
```

追加後 `source ~/.zshrc` を実行。

**方法3：インストール状態の点検**

```bash
claude doctor
```

## Q2. ログインできない

### 症状

- ブラウザが開かない
- 認証画面でフリーズする
- ログイン後も認証エラーが出る

### 解決法

**ブラウザポップアップが開かないとき**

`claude` 実行後に `c` を押して認証URLをコピーし、ブラウザに直接貼り付けてください。

**ログアウト後に再認証**

```bash
# Claude Code内で
> /logout

# または認証ファイルを直接削除
rm -rf ~/.config/claude-code/auth.json
claude
```

**それでもダメな場合**

```bash
# すべての設定をリセットして再起動
rm ~/.claude.json
rm -rf ~/.claude/
claude
```

> 注意：この操作を行うと、すべての設定とセッション履歴が削除されます。

## Q3. 動作が遅い / 応答が遅い

### 原因

会話が長くなるにつれて、Claudeが処理すべき内容（コンテキスト）が蓄積され、遅くなります。これは正常な動作です。

### 解決法

**方法1：`/compact` でコンテキストを整理（最も効果的）**

```
> /compact
```

会話内容を圧縮して速度を回復します。

**方法2：`/clear` で初期化**

完全にやり直しても問題ない場合：

```
> /clear
```

**方法3：より高速なモデルに変更**

複雑な作業でなければ、HaikuやSonnetはOpusよりはるかに高速です。

```
> /model
```

モデル選択画面で変更してください。

**方法4：MCPサーバーを減らす**

不要なMCPサーバーが多いと遅くなることがあります。

```
> /mcp
```

使用していないサーバーを無効化してください。

## Q4. 思い通りに修正してくれない

### 原因

指示が曖昧、Claudeがプロジェクトの文脈を理解していない、または同じ話を繰り返してClaudeが混乱している場合です。

### 解決法

**方法1：より具体的に指示する**

```
# 曖昧な指示
> コードを改善して

# 具体的な指示
> auth.jsのlogin関数でメールのバリデーションが抜けています。
  正規表現でメール形式を確認するコードを42行目の後に追加してください
```

**方法2：CLAUDE.mdにルールを追加**

繰り返し発生する要件はプロジェクトルールとして保存しましょう。

```
> /memory
```

またはCLAUDE.mdファイルに直接追加：

```markdown
# コードスタイル
- エラーメッセージは常に日本語で記述
- 関数ごとにJSDocコメント必須
- varの使用禁止、const/letのみ使用
```

**方法3：同じミスが2回繰り返されたら初期化**

```
> /clear
```

新しく始めて、より明確な指示で再度リクエストしてください。

## Q5. ファイルを修正したがミスだったようだ

### 解決法

**方法1：Claude Code内で巻き戻し（最も速い）**

```
Escキーを2回連続で押す
または
> /rewind
```

以前の状態に会話とコードをまとめて戻します。

**方法2：gitで復旧**

```bash
# 変更内容の確認
git diff

# 特定のファイルだけ元に戻す
git checkout -- src/auth/login.js

# すべての変更を取り消し
git checkout .
```

**方法3：作業前のgit commit習慣を身につける**

Claude Codeで大きな作業を始める前に必ず：

```bash
git add .
git commit -m "作業前バックアップ $(date +%Y%m%d-%H%M)"
```

## Q6. 突然コストが高くなった

### 原因

- 会話が長くなりすぎた（コンテキストが大きいほどトークン消費が増加）
- ファイルの多いプロジェクト全体の分析を依頼
- Opusモデルを使い続けた
- エージェントチーム機能の使用

### 解決法

**現在のコストを確認**

```
> /cost
```

API利用者向け。サブスクリプション利用者は `/stats` で使用量を確認。

**コストを削減する方法**

1. `/compact` を頻繁に使ってコンテキストを管理
2. 単純な作業にはSonnetやHaikuを使用（`/model` で変更）
3. 大きなファイル/フォルダ全体を文脈なく読み込ませない
4. 関連のない作業の間には `/clear` で初期化

**予算の上限設定（自動化使用時）**

```bash
claude -p --max-budget-usd 1.00 "質問"
```

## Q7. 権限リクエストが多すぎる

### 状況

Claudeがファイルを修正したりコマンドを実行するたびに許可を求めてきます。

### 解決法

**方法1：よく使うツールは常に許可に設定**

```
> /permissions
```

権限管理画面で特定のツールをAlways Allowに設定します。

**方法2：ファイル修正を自動許可**

```
Shift+Tabを押してAuto-Accept Edit Modeを選択
```

現在のセッション中、ファイル修正リクエストを自動的に許可します。

**方法3：各権限モードを理解する**

| モード | 説明 | 使うタイミング |
|------|------|----------|
| デフォルトモード | 初回使用時に許可を要求 | 通常使用 |
| Plan Mode | ファイル修正不可、計画のみ | 安全に調査するとき |
| Auto-Accept | ファイル修正を自動許可 | 信頼できる作業 |

**`--dangerously-skip-permissions` の使用に注意**

このオプションはすべての権限確認をスキップします。隔離されたコンテナ環境でなければ使用しないでください。誤ってシステムファイルを修正・削除するリスクがあります。

## Q8. 日本語が文字化けする / 文字が正しく表示されない

### 原因

ターミナルがUTF-8エンコーディングを使用していない場合に発生します。

### 解決法

**macOS / Linux**

シェル設定ファイル（`.zshrc` または `.bashrc`）に追加：

```bash
export LANG=ja_JP.UTF-8
export LC_ALL=ja_JP.UTF-8
```

追加後 `source ~/.zshrc` を実行。

**ターミナルアプリの設定確認**

- **iTerm2**：Preferences → Profiles → Terminal → Character Encoding → UTF-8
- **macOS標準ターミナル**：環境設定 → プロファイル → 詳細 → 文字エンコーディング → Unicode(UTF-8)
- **VS Codeターミナル**：自動的にUTF-8を使用（ほとんどの場合問題なし）

**Windows（WSL）**

```bash
# WSLで実行
sudo locale-gen ja_JP.UTF-8
```

## Q9. コンテキストを失った（セッションが初期化された）

### 状況

- Claudeが以前の作業を覚えていない
- セッションが突然切れた
- ターミナルを閉じて再度開いた

### 解決法

**方法1：前回のセッションを引き継いで開始**

```bash
# 最新のセッションを引き継ぐ
claude --continue

# セッション一覧から選択
claude --resume

# 名前で引き継ぐ（事前に名前を保存した場合）
claude --resume 競合リサーチ-2月
```

**方法2：セッションに名前を付けておく（予防策）**

重要な作業を始めるとき：

```
> /rename 競合リサーチ-2月
```

**方法3：CLAUDE.mdに重要情報を保存**

セッションが切れても維持すべき情報はCLAUDE.mdに：

```
> /memory
```

```markdown
# 現在進行中の作業
- 目標：Q1市場競合分析レポートの完成
- 完了：A社、B社の分析
- 進行中：C社の分析、最終比較表の作成
- 参考ファイル：research/competitors/ フォルダ
```

## Q10. コードが大幅に変わってしまった（予期しない大規模変更）

### 状況

「簡単に修正して」と言ったのに、Claudeが関係ないファイルまで大量に修正してしまうケース。

### 解決法

**方法1：すぐに止める**

Claudeが作業中であれば：

```
Ctrl+C
```

または `Esc` を押して現在の作業を中断。

**方法2：以前の状態に巻き戻す**

```
Escを2回連続
または
> /rewind
```

**方法3：gitで確認して復旧**

```bash
# 変更されたファイル一覧を確認
git status

# 変更内容の詳細確認
git diff

# 特定のファイルだけ復旧
git checkout -- ファイルパス

# すべて取り消し
git checkout .
```

**方法4：Plan Modeで事前確認（予防策）**

大きな作業の前にはPlan Modeで計画を先に確認しましょう。

```
Shift+Tab（2回）
```

または

```bash
claude --permission-mode plan
```

**方法5：指示をより具体的に**

```
# 範囲を明確に指定
> auth.jsファイルのlogin関数だけ修正して。他のファイルは触らないで

# 禁止事項を明示
> リファクタリングはせず、バグだけ修正して
```

## Q11. Claudeがドキュメントの一部だけでなく全体を書き直してしまった

### 状況

「3段落目だけ直して」と言ったのに、Claudeがドキュメント全体を最初から書き直してしまうケース。

### 解決法

**方法1：すぐに巻き戻す**

```
Escを2回連続
または
> /rewind
```

**方法2：範囲をより明確に指定する**

```
# 曖昧な指示
> 3段落目を直して

# 具体的な指示
> 「導入部の2番目の段落で『しかし』で始まる文だけをもう少し柔らかい表現に変えてください。

```

**方法3：Plan Modeで事前確認**

`Shift+Tab` でPlan Modeに入る → Claudeが「どこをどのように変更する予定か」を先に表示します。確認後に承認。

## Q12. 日本語だけで検索したら情報が不足している

### 状況

Claudeにリサーチを依頼したが、日本語の資料しか探さず結果が乏しいケース。

### 解決法

検索言語を明示的に指示してください：

```
「英語、日本語の資料を両方検索して整理してください」

「主に英語の資料を探して、核心的な内容は日本語で要約してください」

「グローバルトレンドは英語資料ベースで、国内の状況は日本語で調査してください」
```

Claudeは言語を指定しないと、日本語での会話の文脈から日本語資料を中心に検索する傾向があります。

## Q13. AIが推奨したパッケージをインストールしたが怪しい 🖥️ 開発者向け

### スロップスクワッティング（Slopsquatting）とは？

AIモデルが実際には存在しないパッケージを自信たっぷりに推奨（ハルシネーション）した際に、攻撃者がその名前で悪意のあるコードを含むパッケージをnpm/PyPIに事前に登録しておく攻撃手法です。

Claudeが `react-auth-helper` というパッケージを推奨したが、実際には存在しないか、同じ名前の悪意あるパッケージが登録されている可能性があります。

### インストール前に必ず確認する方法

**npmパッケージ情報を先に確認**

```bash
# インストール前にパッケージ情報を確認
npm info パッケージ名

# ダウンロード数、最終更新日、依存関係を確認
# → 公開されて数日しか経っていない、またはダウンロード数が極端に少ない場合は注意
```

**Claudeに出典も一緒に聞く**

```
「このパッケージが本当に存在するか、npmjs.comのリンクと
GitHubリポジトリのURLも一緒に教えてください」
```

**見慣れないパッケージよりも有名なものを優先**
- 似た機能であればGitHub starが多く、歴史の長いパッケージを選択
- Anthropic、Microsoft、Vercel、Metaなど信頼できる組織のパッケージを優先
- 初めて見るパッケージはREADMEとソースコードを直接確認

**インストール後のコードレビュー**

```bash
# インストールされたパッケージのエントリーポイントを確認
cat node_modules/パッケージ名/index.js | head -100
```

> 💡 **docs-guide**プラグインを使用すると、Claudeが公式ドキュメントを直接参照するため、存在しないパッケージを推奨する可能性が大幅に減少します。

## Q14. Claude作業後にセキュリティが心配 / 予期しない変更が発生した 🖥️ 開発者向け

### AIコードをマージする前に確認すべき8つのチェックポイント

Claudeが作成または修正したコードをコミットする前に確認してください。

```
セキュリティチェックリスト：
□ 1. ハードコードされたシークレットはないか？（APIキー、パスワード、トークン）
□ 2. ユーザー入力が適切に検証されているか？（SQLインジェクション、XSS防止）
□ 3. 動的コード実行関数を使用していないか？（eval系）
□ 4. ファイルパスがユーザー入力をそのまま使用していないか？（パストラバーサル）
□ 5. エラーメッセージに内部構造が露出していないか？
□ 6. 外部URLリクエスト時にドメイン検証があるか？（SSRF防止）
□ 7. 依存関係（パッケージ）が突然追加されていないか？
□ 8. 既存の認証・権限ロジックがバイパスされていないか？
```

### `git add -p` で変更内容を行単位でレビュー

`git add .` はClaudeが変更したすべてを一括でステージングします。代わりに `git add -p` を使えば、変更のチャンク（hunk）ごとに含めるかどうかを選択できます。

```bash
git add -p

# 各変更チャンクで：
# y → この変更を含める
# n → この変更を除外
# s → さらに小さく分割して表示
# d → このファイルの残りをすべて除外
# q → 終了
```

### 疑うべきAI作業パターン

| パターン | 注意すべき理由 |
|------|----------|
| 依頼していないファイルも修正されている | 意図しないサイドエフェクトの可能性 |
| コメントなしで既存コードを丸ごと置換 | 何が変わったか把握が困難 |
| 設定ファイル（`.env`、`package.json`）が突然修正 | 環境変数・依存関係の変化を要チェック |
| テストファイルだけ削除または無効化 | 既存の検証をバイパスする可能性 |
| 見慣れないパッケージの追加 | スロップスクワッティングのリスク（Q13参照） |
| エラー処理を空のcatchブロックで包む | 例外が静かに消える |

> **原則**：Claudeが生成したコードも、外部から受け取ったコードと同様にレビューしてください。AIが作成したからといって安全が保証されるわけではありません。

## まとめ：よくあるミス防止チェックリスト

```
Claude Code開始前：
□ プロジェクトフォルダに移動したか？（cd myproject）
□ 重要な作業ならgit commitしたか？
□ CLAUDE.mdにプロジェクトルールがあるか？

作業中：
□ 指示が十分に具体的か？
□ 一度に多くを要求しすぎていないか？
□ 会話が長くなったら/compactを使ったか？
□ 何かおかしければCtrl+Cですぐに止めたか？

作業後：
□ 変更内容を確認したか？（git diff）
□ テストが通っているか？
□ 重要なセッションなら名前を保存したか？（/rename）
```
