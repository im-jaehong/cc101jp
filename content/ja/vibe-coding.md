# 03. バイブコーディングとは？

> コードを知らない人がAIに自然言語で指示してソフトウェアを作る新しい方法論

---

## バイブコーディング（Vibe Coding）とは？

「雰囲気（vibe）でコーディングする」という意味です。Andrej Karpathy（元Tesla AI責任者）が2025年に名付けた概念で、**コードを自分で書かずにAIに自然言語で指示してソフトウェアを作る方式**です。

従来のコーディング: 文法を覚え、エラーを自分で修正し、ライブラリを学習する必要がある
バイブコーディング: 「ログイン機能を作って」→ AIがコードを生成 → 結果を見てフィードバック

**Claude Codeはバイブコーディングに最も適したツールの一つです。** ターミナルでファイルを直接読み込み、修正し、実行し、テストするところまですべてAIが処理します。

---

## バイブコーダー vs 開発者

| | 開発者 | バイブコーダー |
|--|--------|-----------|
| コード作成 | 自分で書く | AIに指示 |
| デバッグ | エラーメッセージを分析 → 修正 | 「これエラーが出るんだけど直して」 |
| 主な役割 | 実装 | **企画＆検収** |
| 必要な能力 | プログラミング言語 | **明確な要件の伝達** |

バイブコーディングで最も重要なのはコーディングスキルではなく、**「何を作りたいのかを明確に説明する能力」**です。

---

## バイブコーディングの現実: メリットと限界

### 本当にできること
- プロトタイプ、MVPの素早い作成
- 個人用ツール、自動化スクリプト
- Webサイト、シンプルなアプリ
- データ処理、ドキュメント自動化
- **このサイト（CC101）もバイブコーディングで作りました**

### 正直な限界
- 大規模な商用サービスには専門の開発者によるレビューが必要
- セキュリティ、決済などのセンシティブなロジックには必ず専門家のレビューが必要
- AIが生成したコードを理解できないとデバッグが難しい
- 「作った」と「保守する」は別の問題

> **ヒント**: バイブコーディングで始めつつ、サービスが大きくなったら開発者と協業するのが賢明です。

## バイブコーディングを上手にやるコツ

### 1. 企画が半分
「アプリ作って」よりも「写真を日付別に整理するWebアプリを作って。ドラッグ＆ドロップでアップロードしたらEXIF情報で自動分類」の方がはるかに良い結果が出ます。

→ **show-me-the-prd**プラグイン: 5〜6回のインタビューで企画書を自動生成してくれます。

### 2. 小さく始めて大きく育てる
一度に完成品を求めないでください。「ログイン画面をまず」→「次はリスト画面」→「検索機能を追加」の順で段階的に作れば、はるかに安定します。

### 3. 結果を自分で確認する
AIが作った成果物を必ず実行して確認してください。「うまくいった」「この部分は違う感じにして」というフィードバックが品質を高めます。

### 4. 失敗を恐れない
Gitで保存しておけばいつでも元に戻せます。失敗しても損することはありません。

→ **バルダ Git先生**プラグイン: Gitを知らない方でも「保存して」の一言でバージョン管理が可能

## バイブコーダーのためのGPTakuプラグイン

Claude Codeにインストールするとバイブコーディングがずっと簡単になるプラグインです:

| プラグイン | 役割 | 一言説明 |
|---------|------|----------|
| **show-me-the-prd** | 企画 | 「アプリ作りたい」→ インタビュー → 4種の企画書を自動生成 |
| **バルダ Git先生** | バージョン管理 | Gitを知らなくても「保存して」「プッシュして」だけでOK |
| **バ先生** | 成長 | AI使用パターン分析、より上手に使う方法のコーチング |
| **docs-guide** | 正確性 | 公式ドキュメントベースの回答でハルシネーション防止 |
| **deep-research** | リサーチ | マルチエージェントで技術調査、市場分析 |
| **끼리끼리** | チーム構成 | 「リサーチチームを作って」→ AIチームを自動構成 |
| **품앗이** | 並列開発 | Claude（PM）がタスクを分担し並列処理。Codexがあれば活用、なければClaudeだけでOK |
| **스킬러들의 수다** | スキル制作 | 「翻訳スキルを作って」→ 4人の専門家が設計して生成 |

インストール方法は**プラグインインストール**セクションで取り上げます。

## バイブコーディングのワークフロー例

### 「タスク管理アプリを作りたい」

```
Step 1: 企画
/show-me-the-prd タスク管理アプリを作りたい
→ 5回のインタビュー → PRD + データモデル + フェーズ分割 + プロジェクトスペック

Step 2: 実装
「Phase 1から作って」
→ Claude Codeがコード生成、ファイル保存、実行まで

Step 3: 確認＆フィードバック
「画面を見せて」→ ブラウザで確認
「ボタンの色を変えて」「日付ソートを追加して」→ 会話で修正

Step 4: 保存
/git-teacher 保存して
→ Gitコミットを自動生成
```
